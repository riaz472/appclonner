# VirtualApp — Android Build Environment

A legacy Android project (VirtualApp) configured to compile fully inside Replit.

## Project Overview

VirtualApp is an Android app that uses:
- **Gradle 7.6.1** with Android Gradle Plugin 7.4.2
- **compileSdkVersion 28** (Android 9 / Pie)
- **NDK r16b** (required — uses `gnustl_static` which was removed in NDK r18+)
- Native JNI code built via `ndk-build` (armeabi-v7a + x86 targets)

## First-Time Setup

Before building for the first time, run the setup script to download the Android SDK, build-tools, and NDK r16b:

```bash
bash setup-android.sh
```

This script will:
1. Download Android command-line tools
2. Accept all SDK licenses automatically
3. Install `platforms;android-28` and `build-tools;28.0.3`
4. Download and install NDK r16b (~800 MB)
5. Write `VirtualApp/local.properties` pointing Gradle to the SDK/NDK

## Building the APK

After setup is complete:

```bash
cd VirtualApp && ./gradlew assembleDebug
```

Or use the **"Build Android APK"** workflow in the Replit UI (runs setup + build together).

The output APK will be at:
```
VirtualApp/app/build/outputs/apk/debug/app-debug.apk
```

## Key Files

- `setup-android.sh` — One-time SDK + NDK setup script
- `VirtualApp/local.properties` — Generated by setup; tells Gradle where the SDK/NDK live
- `VirtualApp/build.gradle` — Root build file (AGP 7.4.2)
- `VirtualApp/app/build.gradle` — App module (compileSdk 28)
- `VirtualApp/lib/build.gradle` — Library module with NDK build
- `VirtualApp/lib/src/main/jni/Android.mk` — NDK build definition
- `VirtualApp/lib/src/main/jni/Application.mk` — NDK ABI/STL config

## Why NDK r16b?

The `Application.mk` specifies `APP_STL := gnustl_static`. The `gnustl` C++ runtime was deprecated in NDK r17 and removed in NDK r18. NDK r16b is the last stable version that supports it, making it required for this project.

## User Preferences

- Keep setup automated and repeatable via `setup-android.sh`
- Do not upgrade NDK unless `APP_STL` is also migrated to `c++_static`
